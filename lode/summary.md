BoxTracker is a single-user home moving assistant for tracking packed boxes and their garage locations. It produces printable QR labels for boxes and locations, records items per box with optional photos, and provides full-text search across all items via SQLite FTS5.

Stack: F# backend (Saturn/Giraffe on ASP.NET Core), F# frontend (Fable 5 + Elmish + Feliz + DaisyUI 5 on Tailwind CSS 4), shared domain types in a netstandard2.0 library. SQLite for persistence, flat files for photos (WebP format with automatic compression). Vite for bundling and dev server. Runs locally from `nix develop` with `dotnet run` (server) and `npm start` (client). UI uses a custom "industrial" dark theme with amber accents, sharp corners, and IBM Plex Mono throughout.

The domain uses an event-sourced move model. Items, boxes, and locations exist as independent entities with no FK columns for placement. Current placement (Container DU: `Unassigned | InBox of BoxId | AtLocation of LocationCode`) is derived from an append-only `move` table — the most recent move per entity determines where it is. Items can be moved between boxes, boxes moved between locations, and any entity unassigned. The Storage layer uses `ROW_NUMBER() OVER (PARTITION BY entity_id ORDER BY moved_at DESC)` to join with the move log. FTS5 search is kept in sync by deriving box/location from moves. Server uses PascalCase JSON to match Fable's record field names. Client DTOs use `array` (not `list`) for JSON collection fields.

Photos are automatically optimized on upload: two WebP variants generated (full-size max 2000×2000, thumbnail max 250×250) at 80% quality. List views load thumbnails for fast rendering; detail views load full-size. Existing photos from before optimization are inaccessible and should be re-uploaded.
